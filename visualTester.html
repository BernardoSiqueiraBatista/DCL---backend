Cadastro POST /auth/register

→ Retorna userId e dispara e-mail com OTP de verificação.

Verificar E-mail POST /auth/verify-otp

→ Cole o código recebido no Gmail (ou veja no terminal do servidor em modo dev).

Login POST /auth/login

→ Retorna accessToken + refreshToken. Tokens salvos automaticamente.

Renovar Token POST /auth/refresh-token

→ Rotaciona o par de tokens. O token antigo é invalidado.

Esqueci a Senha POST /auth/forgot-password

→ Resposta genérica (anti-enumeração). OTP de reset vai para o e-mail.

Redefinir Senha POST /auth/reset-password

→ Usa email (não userId). Revoga todos os refresh tokens após reset.

Logout POST /auth/logout

→ Revoga o refresh token no banco. Operação idempotente.

Estado Local

Tokens e User ID são persistidos no localStorage entre recarregamentos.
